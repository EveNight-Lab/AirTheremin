# 작업 일지

이 디렉토리는 프로젝트의 일일 작업 내용을 기록하는 곳입니다.

## 사용 방법

### 자동 생성

```bash
# 오늘 날짜의 작업 일지 생성/업데이트
npm run work-log

# 특정 날짜의 작업 일지 생성/업데이트
npm run work-log -- 2025-01-15
```

### 수동 작성

작업 일지 파일은 `YYYY-MM-DD.md` 형식으로 저장됩니다. 예: `2025-01-15.md`

## 작업 일지 구조

각 작업 일지는 다음 섹션으로 구성됩니다:

- **📋 작업 개요**: 오늘의 주요 작업 목표
- **✅ 완료된 작업**: Git 커밋을 기반으로 자동 생성되며, 수동으로 추가 가능
- **📝 변경된 파일**: Git 로그를 기반으로 자동 생성
- **🚧 진행 중인 작업**: 아직 완료되지 않은 작업
- **📅 다음 작업 계획**: 다음에 할 작업 목록
- **💡 이슈 및 메모**: 발생한 문제나 특이사항
- **⏰ 작업 시간**: 작업 시작/종료 시간 및 총 작업 시간

## 주의사항

- 작업 일지는 Git 커밋 로그를 기반으로 자동으로 생성됩니다
- 기존 내용은 유지되며, 새로운 커밋 정보만 추가됩니다
- 필요시 수동으로 내용을 수정할 수 있습니다
- 작업 일지 파일은 Git에 커밋하는 것을 권장합니다
